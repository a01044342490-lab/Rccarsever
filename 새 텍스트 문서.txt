3단계: GitHub에 올리기

이제 이 Rccarserver 폴더를 깃으로 올려야 Railway가 가져갈 수 있어.

PowerShell에서(여전히 Rccarserver 폴더 안):

git init
git add .
git commit -m "Init RC car server"
git branch -M main
git remote add origin https://github.com/네아이디/rc-car-node-server.git
git push -u origin main


GitHub에서 New repository 하나 먼저 만들어 두고(예: rc-car-node-server) 그 주소를 origin에 넣으면 됨.

4단계: Railway에서 배포

브라우저에서 https://railway.app
 접속

Sign in with GitHub 선택 → 로그인

대시보드에서 “New Project” → “Deploy from GitHub repo” 클릭

방금 만든 rc-car-node-server 저장소 선택

조금 기다리면 빌드/Deploy 끝나고 프로젝트에 초록불 뜸

포트 & start 스크립트는 이미 준비됨

package.json에 이게 들어있어야 해:

"scripts": {
  "start": "node app.js"
}


app.js 안에는:

const PORT = process.env.PORT || 3000;
...
app.listen(PORT, () => {...});


이렇게 되어 있으면 Railway가 자동으로 포트/실행 둘 다 알아서 처리함.

5단계: Railway 주소 확인

Railway 프로젝트 화면 들어가면
상단에 “Open URL” 버튼이 있을 거야.

예를 들어:

https://rccarserver-production.up.railway.app


이 주소가 바로 전 세계 어디서든 접속 가능한 서버 주소.

이 주소로 브라우저에서 들어가면 → 웹 로그 화면이 떠야 하고

/data로 들어가면 JSON이 보일 거야.

6단계: ESP-01 / ESP32에서 이 주소로 업로드

ESP-01 코드에서 이 줄을:

const char* serverURL = "http://YOUR_SERVER_IP:3000/upload";


Railway 주소 기준으로 이렇게 바꿔 줘:

const char* serverURL = "https://rccarserver-production.up.railway.app/upload";


이제:

Mega → ESP-01으로 JSON 전송

ESP-01 → Railway /upload로 POST

브라우저에서 https://rccarserver-production.up.railway.app 접속하면
RC카 데이터가 실시간으로 쌓여 보이게 되는 구조야.
